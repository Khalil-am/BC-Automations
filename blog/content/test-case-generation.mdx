## Automation Name

**Test Case Generation Automation**

This automation generates comprehensive test case documents for User Acceptance Testing (UAT), System Integration Testing (SIT), and regression testing. Perfect for consultants managing ERP implementations, CRM rollouts, or any software deployment project.

---

## Prompt

```text
You are a senior QA consultant specializing in enterprise software testing.
Generate a comprehensive test case document for [MODULE/FEATURE NAME]
in [SYSTEM NAME].

The test case document should include:
1. Document Header — project name, module, prepared by, date, version
2. Test Scope — what is being tested and what is out of scope
3. Test Environment — required environment setup and test data
4. Test Cases in table format with these columns:
   - Test Case ID (format: TC-[MODULE]-001)
   - Test Scenario
   - Test Steps (numbered)
   - Test Data (specific values to use)
   - Expected Result
   - Actual Result (leave blank for execution)
   - Status (Pass/Fail — leave blank)
   - Priority (High/Medium/Low)
   - Remarks
5. Include test cases for:
   - Positive scenarios (happy path)
   - Negative scenarios (invalid inputs, boundary values)
   - Edge cases
   - Permission/role-based access testing
   - Data validation and integrity checks
6. Test Summary section with total test cases by priority
7. Sign-off section with fields for Tester, Date, and Reviewer

Generate at least 15 test cases covering all scenario types.
Use a professional format suitable for enterprise UAT documentation.
```

---

## Description

The Test Case Generation Automation eliminates the tedious process of writing test cases from scratch. As a business consultant, you often need to prepare UAT documents for client sign-off during system implementations. This automation produces structured, professional test case documents that follow industry-standard formats.

### Key Benefits

- **Comprehensive Coverage** — Automatically includes positive, negative, and edge cases
- **Standardized Format** — Consistent test case ID naming and table structure
- **Role-Based Testing** — Built-in prompts for permission and access testing
- **Client-Ready** — Professional format suitable for UAT sign-off documentation
- **Traceable** — Each test case has a unique ID for requirement traceability

### How to Use

1. Copy the prompt above
2. Replace `[MODULE/FEATURE NAME]` with the specific feature (e.g., "Purchase Order Creation")
3. Replace `[SYSTEM NAME]` with the target system (e.g., "SAP S/4HANA")
4. Run the prompt in your preferred AI tool
5. Review generated test cases against actual system behavior
6. Add actual test data relevant to the client environment
7. Export and distribute for UAT execution

### Test Case Categories

| Category | Description | Example |
|----------|-------------|---------|
| Positive | Happy path scenarios | Create PO with valid data |
| Negative | Invalid input handling | Submit form with empty required fields |
| Boundary | Edge value testing | Enter maximum character limit |
| Security | Role-based access | Access module without proper authorization |
| Data | Data integrity checks | Verify calculations and data persistence |
