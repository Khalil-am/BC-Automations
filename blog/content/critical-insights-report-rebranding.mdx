## Automation Name

**Critical Insights Report Rebranding — Multi-Format (PPTX / DOCX)**

This automation takes a data-heavy source report (Arabic or English) and rebrands it into a client's official template with their color scheme, fonts, tables, and layout. Instead of manually rebuilding every table and section in a new template, this prompt produces a fully formatted, presentation-ready report in minutes — covering all data sections with consistent client branding and professional table styling.

---

## Prompt

```text
I have two files:

1. A source report (.docx) containing project data with multiple sections:
   - Project performance tables
   - Financial performance tables
   - Delayed tasks tables
   - Challenges/risks tables
   - Cover page and table of contents

2. A client template (.docx) with their official branding (logo, colors, fonts, layout)

Please:
1. Read the source report and extract ALL data from every table and section
2. Read the client template to identify:
   - Color scheme (primary, secondary, accent colors)
   - Font family used in headings and body text
   - Logo and background images
   - Page layout (orientation, margins, header/footer style)
3. Create a new [FORMAT: pptx / docx] report that:
   - Uses the client template's branding (colors, fonts, logo, backgrounds)
   - Contains ALL data from the source report — no data should be missing
   - Structures the content as:
     a. Cover page with report title and export date
     b. Table of contents listing all sections
     c. Each data section with:
        - Section title styled in client's heading format
        - Professional data tables with:
          * Client's primary color as header row background
          * White text on header rows
          * Alternating row shading for readability
          * Status indicators with colored text (red for delayed, green for active, orange for high priority)
          * Proper RTL alignment for Arabic content
     d. Thank you / closing page with client branding
   - All text alignment matches the content language (RTL for Arabic, LTR for English)
4. Run a quality check after generation:
   - Verify all sections from source are present
   - Verify all table rows match the source data count
   - Verify color scheme matches client template
   - Verify fonts are consistent throughout
5. Output as a single .[FORMAT] file
```

## Description

Organizations frequently need to rebrand data reports when delivering to different clients or stakeholders. A source report exported from a system like Infath, Oracle, or SAP contains raw data in a generic format. The client expects that same data presented in their official template with their logo, colors, and fonts.

This automation handles the entire rebranding process: extracting every table, metric, and section from the source, then rebuilding it in the client's template format with professional styling.

### What Gets Generated

A complete report with all source data, rebranded in the client's style:

| Section | Content | Styling |
|---------|---------|---------|
| Cover Page | Report title, export date, client logo | Client's background image, primary color, branded fonts |
| Table of Contents | Numbered list of all sections | Navy header bar, clean numbered entries |
| Project Performance | All project rows with department, type, and % complete | Navy header row, alternating row shading, centered data |
| Financial Performance | Budget, expenditure, planned vs actual for each project | Status column in red with colored indicators |
| Delayed Tasks | Task name, owner, dates, completion %, status | Red status text, alternating rows for readability |
| Challenges | Risk name, owner, due date, classification, status | Orange for high priority, green for active status |
| Closing Page | Client logo, thank you message | Full client branding |

### Color Scheme Extraction

Claude automatically identifies the client's color scheme from their template:

| Element | Source (Infath) | Target (MHC) | Applied To |
|---------|----------------|--------------|------------|
| Primary | #13778D (teal) | #070E4F (navy) | Headings, table headers, title text |
| Accent | #17A3A3 (turquoise) | #1B3A8C (blue) | Sub-headings, accent lines |
| Background | White | White with #F0F3FA alternating rows | Table rows, page background |
| Status: Delayed | Red text | #C0392B (red) | Status indicators |
| Status: Active | Green text | #27AE60 (green) | Active status indicators |
| Status: High Priority | — | #E67E22 (orange) | Priority classification |
| Font | System default | Tajawal | All text throughout |

### How It Works

1. **You provide** a source report (.docx) — this is your data
2. **You provide** the client's branded template (.docx) with their logo, colors, and fonts
3. **Claude reads** the source report and extracts all tables, sections, and data points
4. **Claude analyzes** the client template to identify: color hex codes, font family, logo image, background images, page layout (landscape/portrait), and margins
5. **Claude builds** a new document mapping every source section into the client's style
6. **Claude applies** professional table formatting: navy header rows, white header text, alternating row shading, colored status indicators
7. **Claude runs QA** to verify all data is present and styling is consistent
8. **Claude outputs** a single file (.pptx or .docx) ready for delivery

### Key Benefits

- **Zero manual formatting** — No copying tables cell-by-cell into a new template
- **Automatic color scheme matching** — Client's exact hex colors applied throughout
- **Font consistency** — Client's font applied to every heading, table, and body text element
- **Data integrity** — Every row from every table transferred with zero data loss
- **Dual format output** — Choose PPTX for presentations or DOCX for printed reports
- **RTL/LTR support** — Arabic and English content properly aligned
- **Professional table styling** — Navy headers, alternating rows, colored status indicators
- **Minutes not hours** — Complete rebranding in a single Claude session (~3-5 minutes)
- **Built-in QA** — Claude verifies section count, data completeness, and styling consistency

## Best Practices

- Always use the **original source report** — do not pre-format or modify it before uploading
- Provide the **cleanest version** of the client template — remove any sample content, leave only branding elements
- Specify the **output format** clearly in the prompt (PPTX or DOCX)
- For **Arabic content**, verify RTL alignment in the output — especially in tables
- If the source has **images or charts**, mention them in the prompt so Claude preserves or recreates them
- For **large reports** (50+ tables), consider splitting into multiple prompts by section
- Always **open the output file** and do a visual review before sending to the client
