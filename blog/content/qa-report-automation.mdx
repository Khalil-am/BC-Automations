## Automation Name

**Quality Assurance Report Automation**

This automation generates professional QA summary reports that consolidate testing progress, defect metrics, and go/no-go recommendations. Designed for project managers and business consultants who need to present testing status to stakeholders and steering committees.

---

## Prompt

```text
You are a senior QA manager preparing a quality assurance status report
for a steering committee presentation.
Generate a comprehensive QA report for [PROJECT NAME] — [TESTING PHASE]
as of [REPORTING DATE].

The report should include:
1. Executive Summary
   - Overall testing status (On Track/At Risk/Delayed)
   - Key achievements this reporting period
   - Critical blockers and risks
   - Go/No-Go recommendation with justification
2. Testing Progress Dashboard (text-based)
   - Total test cases: [NUMBER]
   - Executed: [NUMBER] ([PERCENTAGE]%)
   - Passed: [NUMBER] ([PERCENTAGE]%)
   - Failed: [NUMBER] ([PERCENTAGE]%)
   - Blocked: [NUMBER] ([PERCENTAGE]%)
   - Not Executed: [NUMBER] ([PERCENTAGE]%)
3. Defect Summary
   - Total defects raised
   - Defects by severity (Critical/High/Medium/Low) in table format
   - Defects by status (Open/In Progress/Resolved/Closed)
   - Defect aging analysis
   - Top 5 critical/high defects with descriptions
4. Module-wise Testing Status — table with:
   - Module name
   - Total TCs
   - Executed
   - Pass rate
   - Open defects
   - Status (RAG: Red/Amber/Green)
5. Risk Register — table with:
   - Risk ID
   - Description
   - Impact (High/Medium/Low)
   - Probability (High/Medium/Low)
   - Mitigation plan
   - Owner
6. Test Environment Status
   - Environment availability
   - Known environment issues
   - Data refresh status
7. Next Steps and Action Items — table with:
   - Action item
   - Owner
   - Due date
   - Status
8. Appendix
   - Defect trend chart description (for visualization)
   - Test execution trend description

Use placeholder values marked with [PLACEHOLDER] for actual metrics.
Format for executive-level presentation.
```

---

## Description

QA reports are a recurring deliverable in every testing engagement. This automation generates a complete, stakeholder-ready QA report with all the sections that steering committees expect — from executive summaries to defect trend analyses.

### Key Benefits

- **Stakeholder-Ready** — Executive summary and RAG status for quick decision-making
- **Comprehensive Metrics** — Defect analysis, module-wise status, and risk registers
- **Actionable** — Built-in action items table with ownership and deadlines
- **Repeatable** — Use the same template weekly throughout the testing phase
- **Decision Support** — Go/No-Go recommendation framework included

### How to Use

1. Copy the prompt above
2. Replace project name, testing phase, and reporting date
3. Run the prompt to generate the report structure
4. Fill in actual metrics from your test management tool
5. Update the defect details from your defect tracking system
6. Review the Go/No-Go recommendation
7. Present to stakeholders or distribute as a status report
